# Product Notes

FoodOps Community is designed for small and mid-sized chain food teams that need a self-hosted operating loop before investing in private integrations.

The first screen should be the actual workbench, not a marketing surface. Keep the UI calm, dense, and operational: data tables, filters, task state, exception priority, and auditability are more important than decorative visuals.

Community V0.1 success means:

- A new developer can start the stack locally.
- Demo data shows the full loop from dashboard to alert to task and feedback.
- A store manager can use the H5 task view without external messaging.
- Admin users can manage master data and inspect audit logs.
- No private brand, customer data, external connector secret, or paid model dependency is required.

Do not add public-opinion monitoring, social media workflows, external AI providers, enterprise messaging, or autonomous agent orchestration to the community core. Keep those as separate Enterprise/plugin work.
