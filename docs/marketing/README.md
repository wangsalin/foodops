# Marketing Kit

This folder contains conservative launch material for FoodOps Community.

The first public push should target developer validation, not broad user acquisition. The goal is to find contributors who can run the project, inspect the architecture, open issues, and help improve the community core.

## Positioning

FoodOps Community is an open-source, self-hosted operations loop for chain food and beverage teams.

It focuses on:

- local master data
- manual business data imports
- dashboard visibility
- rule-based alerts
- alert-to-task handling
- H5 task feedback
- notifications
- audit logs

It does not claim to replace large local-service platforms, delivery platforms, ERP systems, or customer-specific enterprise integrations.

## First Wave Goal

Within the first public wave, aim for:

- 5 to 10 developers or operators trying the repository locally
- 3 to 5 actionable issues from real setup attempts
- 1 to 2 outside contributors willing to pick up a good first issue
- clearer feedback on import templates, demo data, and local setup

## Recommended Channel Order

1. Private developer and SaaS groups
2. WeChat Moments or personal account post
3. GitHub issue-based contributor invitation
4. Juejin, V2EX, Zhihu, or similar Chinese developer communities
5. English channels only after the local setup and screenshots improve
