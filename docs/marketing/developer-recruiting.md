# Developer Recruiting Copy

## 中文长版

我们刚开源了 FoodOps Community，一个面向连锁餐饮门店的自托管运营系统。

它不是外卖平台，也不是大而全 ERP。社区版只聚焦一条基础运营链路：

`主数据 -> 手动导入 -> dashboard -> 本地规则预警 -> 任务派发 -> H5 反馈 -> 通知与审计`

技术栈比较常规：

- FastAPI
- SQLAlchemy / Alembic
- PostgreSQL
- Redis
- Next.js
- Ant Design

现在最需要的是第一批开发者和行业使用者帮忙把社区版打磨扎实，尤其是：

- 导入模板
- demo seed
- 后端测试
- 前端 smoke tests
- 任务和 H5 反馈流程
- 文档和本地启动体验

GitHub:

https://github.com/wangsalin/foodops

适合参与的人：

- 做过门店运营、餐饮 SaaS、零售系统的人。
- 熟悉 FastAPI / Next.js / PostgreSQL 的开发者。
- 想参与一个真实行业开源项目的人。
- 愿意从小 issue 开始贡献的人。

如果你愿意参与，可以先从 `good first issue` 开始。我们更欢迎具体反馈：哪里跑不起来、字段哪里不合理、任务流哪里不符合门店现场。

一个面向连锁餐饮门店的自托管运营系统，聚焦本地数据导入、dashboard、规则预警、任务派发、H5 反馈和审计日志。

GitHub:

## English Short Version

FoodOps Community is now open source.

It is a self-hosted operations loop for chain food and beverage teams, focused on local imports, dashboards, rule-based alerts, task dispatch, H5 feedback, notifications, and audit logs.

Stack: FastAPI, Next.js, PostgreSQL, Redis.

We are looking for early developers and operators to help improve import templates, demo data, backend tests, frontend smoke tests, and the core alert-to-task workflow.

GitHub:

https://github.com/wangsalin/foodops
