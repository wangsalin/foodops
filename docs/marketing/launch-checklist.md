# Launch Checklist

Use this checklist for the first public wave.

## Before Posting

- [ ] Confirm README renders correctly on GitHub.
- [ ] Confirm CI is green.
- [ ] Confirm `main` branch is protected.
- [ ] Confirm labels and first issues are visible.
- [ ] Confirm no private customer data, logs, uploads, browser profiles, or secrets exist in the repository.
- [ ] Run the project locally from a clean checkout if time allows.
- [ ] Capture real screenshots after demo data is improved.

## First Wave

- [ ] Send the short post to 5 to 10 trusted developers first.
- [ ] Ask them to record setup friction and unclear docs.
- [ ] Ask at least 2 people to try a good first issue.
- [ ] Collect feedback before posting to larger public channels.

## Public Channels

- [ ] WeChat Moments
- [ ] Developer groups
- [ ] Juejin
- [ ] V2EX
- [ ] Zhihu
- [ ] GitHub Discussions or pinned issue after discussion is enabled

## What To Track

- Stars
- Forks
- Issues opened
- Pull requests opened
- Setup failures
- Repeated questions
- Which good first issue is most approachable

## Response Rules

- Answer setup questions quickly.
- Convert repeated questions into docs.
- Do not accept enterprise connector code directly into the core.
- Move enterprise integration ideas into plugin-boundary discussions.
- Keep first-time contributor tasks small.

## Follow-Up Work

- Add import templates.
- Add real screenshots.
- Improve demo seed.
- Add tests for import normalizers.
- Add tests for alert-to-task flow.
- Add frontend smoke tests.
